“O poder da web está em sua universalidade. O acesso de todos, independentemente da deficiência, é um aspecto essencial” - Tim Berners-Lee

Toda vez que um programador opta por fazer um site de qualquer jeito, se importando somente com aparência, cai um fio de cabelo de Tim Berners-Lee. Brincadeiras à parte, o pai da internet sempre imaginou a web como algo libertador, sem fronteiras. As informações devem ser fáceis de serem acessadas. Parece uma coisa óbvia…

Em um primeiro momento, e já peço perdão pela honestidade, o desenvolvedor médio está preocupado em fazer um site bonito, responsivo, cheio de features. Mas por descuido, acabamos nos esquecendo de algumas coisas. O principal é:

Será que meu usuário consegue interagir com meu site? Parece uma pergunta, a primeira vista, muito boba. Mas, atras dela, existe algo muito mais além... É possivel interagir com meu site somente pelo teclado? Parece algo de outro mundo, não é verdade? E se o usuario, por exemplo, interage com o teclado usando uma "caneta" entre os dentes? Ou vamos pensar num tetraplegico como, por exemplo o Stephen Hawking.

Outro tipo nde problema comum são com pessoas com dificulade de visão. Isso pode ir desde o "vovozinho" que não enxerga as letras por serem muito miudinha, até o daltonico, que vai ter dificuldades em diferenciar algumas cores. Tem o caso do "ceguinho" também. Esse é um caso um pouco mais obvio. Com certezq vc já se deparaou, em alguma rede social com a campanha "Para cego ver", onde acrescentamos uma breve descrição sobre o que se trata a imagem de uma figura. Algo como, por exemplo:

Para cego ver: Foto de crianças jogando futebol ao redor de uma casa na fazenda.

Os padrões de acessibilidade vem atender a essa população. Aqui no jonastorres.dev estamos seguindo as recomendações contidas WCAG (Web Content Accessibility Guidelines), do W3C (World Wide Web Consortium), principal organização de padronização da World Wide Web, que desenvolve especificações técnicas e orientações para o mercado. Também, na medida do possível, estamos implementando as orientações do eMAG (O Modelo de Acessibilidade em Governo Eletrônico). O último é voltado normalmente para sites do governo brasileiro, porém, só pelo aprendizado já vale a adoção também.

Nosso site conta com:

  • Barra de acessibilidade;
  • Botões de direcionamento as principais partes de interesse no site;
  • Controle de tamanho de fonte;
  • Ajustes de temas visuais do site;
  • Mapa do site;
  • Botão de direcionamento para o topo;
  • Atalhos de teclado.

Barra de acessibilidade

No topo de nossa página há uma pequena barra que contendo:

  • Links de direcionamento para as principais regiões da página;
  • Link para essa página;
  • Botões para controle de tamanho de fonte;
  • Botões para seleção de tema do site (claro, escuro e alto-contraste);
  • Controle de animações;
  • Mapa do site.

Atalhos de teclado

Os atalhos de teclado tem por finalidade facilitar a navegação sem a utilização do mouse. Infelizmente, cada conjunto de sistema operacional e navegadores, implementaram seus padrões. De modo geral, irei utilizar a nomenclatura [Conjunto de teclas] para destacar a parte que pode mudar de um navegador para outro.

Aqui em jonastorres.dev é utilizado os seguintes atalhos:

[Conjunto de teclas] + 1
Direciona ao cabeçalho da página
[Conjunto de teclas] + 2
Direciona ao conteúdo principal
[Conjunto de teclas] + 3
Direciona ao rodapé

Conhecendo as implementações nos sistemas Windows / Linux

Internet Explorer e Google Chrome

Na lista apresentada acima, devemos ler o [Conjunto de teclas] como Alt.

Por exemplo, o atalho para o cabeçalho da página seria "Alt + 1".

Firefox

Na lista apresentada acima, devemos ler o [Conjunto de teclas] como a combinação das teclas "Alt + Shift".

Por exemplo, o atalho para o conteúdo principal da página seria "Alt + Shift + 2".

Opera

Na lista apresentada acima, devemos ler o [Conjunto de teclas] como a combinação das teclas "Shift + Esc".

Por exemplo, o atalho para o conteúdo principal da página seria "Shift + Esc + 2".

Safari e OmniWeb

Na lista apresentada acima, devemos ler o [Conjunto de teclas] como Ctrl.

Por exemplo, o atalho para o conteúdo principal da página seria "Ctrl + 2 ".

Mac OS

Na lista apresentada acima, devemos ler o [Conjunto de teclas] como a combinação das teclas "Ctrl + Option".

Por exemplo, o atalho para o rodapé da página seria ""Ctrl + Option" + 3".
